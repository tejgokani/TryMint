# Contributing to TRYMINT

> Contribution Guidelines

---

## 📋 Overview

Thank you for your interest in contributing to TRYMINT! This guide helps you get started.

---

## 🏁 Getting Started

1. Fork the repository
2. Clone your fork
3. Set up development environment (see [Development Setup](docs/setup/development.md))
4. Create a feature branch

---

## 📝 Pull Request Process

1. **Create a branch**
   - Feature: `feature/description`
   - Bugfix: `fix/description`
   - Docs: `docs/description`

2. **Make your changes**
   - Follow code style guidelines
   - Add tests if applicable
   - Update documentation

3. **Test your changes**
   - Run `pnpm lint`
   - Run `pnpm typecheck`
   - Run `pnpm test`

4. **Submit PR**
   - Clear description of changes
   - Reference related issues
   - Request review

---

## 🎨 Code Style

- TypeScript for all code
- ESLint for linting
- Prettier for formatting
- Meaningful variable names
- Comments for complex logic

---

## 📁 Project Structure

Familiarize yourself with:

- `frontend/` - Web application
- `backend/` - API server
- `agent/` - Local CLI agent
- `docs/` - Documentation

---

## 🔐 Security

- Never commit secrets
- Follow security guidelines
- Report vulnerabilities privately
